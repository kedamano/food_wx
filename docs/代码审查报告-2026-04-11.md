## 🚨 一、安全问题（严重程度分级）

### 🔴 P0 - 高危（必须修复）

#### 1. 【越权漏洞】订单状态可被任意用户修改
**位置**: `OrderController.java` 第160-176行 `updateOrderStatus`

**问题**: 虽然代码有验证订单归属的逻辑，但用户可以将自己的订单从"已取消"改为"待付款"等任意状态

**修复方案**: 添加状态转换校验

```diff
+ private static final Map<String, List<String>> VALID_STATUS_TRANSITIONS = new HashMap<>();
+ static {
+     VALID_STATUS_TRANSITIONS.put("待付款", Arrays.asList("已取消"));
+     VALID_STATUS_TRANSITIONS.put("制作中", Arrays.asList("已取消"));
+     VALID_STATUS_TRANSITIONS.put("配送中", Arrays.asList("已完成"));
+ }
+
  @PutMapping("/{orderId}/status")
  public BaseResult updateOrderStatus(@PathVariable Integer orderId, @RequestBody StatusRequest statusRequest, HttpServletRequest request) {
      Integer userId = getUserIdFromRequest(request);
      if (userId == null) {
          return BaseResult.error(401, "未登录");
      }
+
+     Order order = (Order) orderService.getOrderById(orderId).getData();
+     if (order == null || !order.getUserId().equals(userId)) {
+         return BaseResult.error(403, "无权操作该订单");
+     }
+
+     String currentStatus = order.getStatus();
+     String newStatus = statusRequest.getStatus();
+     List<String> validNextStatuses = VALID_STATUS_TRANSITIONS.get(currentStatus);
+     if (validNextStatuses == null || !validNextStatuses.contains(newStatus)) {
+         return BaseResult.error(400, "不允许的状态转换: " + currentStatus + " -> " + newStatus);
+     }
+
      return orderService.updateOrderStatus(orderId, statusRequest.getStatus());
  }
```

#### 2. 【垂直越权】管理员接口无权限验证
**位置**: `UserController.java` 第410-449行

**问题**: `GET /user/all` 和 `DELETE /user/{userId}` 未验证用户角色

**修复方案**: 添加角色校验

#### 3. 【敏感信息泄露】用户手机号可被遍历获取
**位置**: `UserController.java` 第269-282行

**修复方案**: 非本人只能获取公开信息
