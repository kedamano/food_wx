## P0级别修复（高危）

### 1. ✅ 订单状态安全校验

**问题描述**: OrderController.updateOrderStatus 缺少状态转换校验，用户可以任意修改订单状态。

**修复内容**:
- 添加订单状态转换白名单 `ALLOWED_STATUS_TRANSITIONS`
- 定义合法的状态流转路径：
  - 待付款 → 已取消、已支付
  - 已支付 → 制作中、已退款
  - 制作中 → 配送中
  - 配送中 → 已完成
- 非法状态转换返回400错误

**涉及文件**:
- `backend/src/main/java/com/food/controller/OrderController.java`

**代码示例**:
```java
private static final Map<String, List<String>> ALLOWED_STATUS_TRANSITIONS = new HashMap<String, List<String>>() {{
    put("待付款", Arrays.asList("已取消", "已支付"));
    put("已支付", Arrays.asList("制作中", "已退款"));
    put("制作中", Arrays.asList("配送中"));
    put("配送中", Arrays.asList("已完成"));
}};
```

### 2. ✅ 管理员接口权限验证

**问题描述**: getAllUsers、deleteUser 等管理接口未验证用户角色，任意用户可调用。

**修复内容**:
- 创建 `@RequireRole` 注解用于标记需要特定角色的接口
- 创建 `RoleCheckAspect` AOP切面统一处理权限验证
- 为管理员接口添加 `@RequireRole("ADMIN")` 注解
- 防止管理员删除自己的账号

**新增文件**:
- `backend/src/main/java/com/food/annotation/RequireRole.java`
- `backend/src/main/java/com/food/aspect/RoleCheckAspect.java`

**修改文件**:
- `backend/src/main/java/com/food/controller/UserController.java`
- `backend/src/main/java/com/food/entity/User.java` (添加role字段)

**代码示例**:
```java
@GetMapping("/all")
@ApiOperation(value = "获取所有用户", notes = "管理员获取所有用户列表")
@RequireRole("ADMIN")
public BaseResult getAllUsers(HttpServletRequest request) {
    // 自动通过AOP验证管理员权限
}
```

### 3. ✅ 敏感信息保护

**问题描述**: UserController.getUserById 返回完整User实体，手机号可被遍历获取。

**修复内容**:
- 创建 `UserDTO` 数据传输对象，用于返回脱敏后的用户信息
- 添加手机号脱敏方法 `maskPhone()`: 13812345678 → 138****5678
- 添加权限验证：只能查看自己的信息，除非是管理员
- 返回DTO而非完整实体

**新增文件**:
- `backend/src/main/java/com/food/dto/UserDTO.java`

**修改文件**:
- `backend/src/main/java/com/food/controller/UserController.java`

**代码示例**:
```java
// 转换为DTO，脱敏敏感信息
UserDTO userDTO = new UserDTO();
userDTO.setUserId(user.getUserId());
userDTO.setPhone(UserDTO.maskPhone(user.getPhone())); // 脱敏
return BaseResult.success(userDTO);
```

## P1级别修复（中危）

### 4. ✅ 积分接口权限控制

**问题描述**: updateUserPoints 接口未限制调用权限，用户可无限增加积分。

- 检查积分是否足够（减少时）

**修改文件**:
- `backend/src/main/java/com/food/controller/UserController.java`

@PutMapping("/points")

public BaseResult updateUserPoints(@RequestBody PointsRequest pointsRequest, HttpServletRequest request) {

### 5. ✅ 验证码安全增强

**问题描述**: verifyCode 验证后未清除验证码，存在重放攻击风险。

**修复内容**:
- 验证码验证成功后立即调用 `clearVerifyCode()` 清除
- 防止同一验证码被重复使用

**修改文件**:
- `backend/src/main/java/com/food/controller/UserController.java`

**代码示例**:
```java
// 验证验证码
if (!com.food.utils.VerifyCodeUtil.verifyCode(phone, code)) {
    return BaseResult.error(400, "验证码错误或已过期");
}

// 验证成功后立即清除验证码，防止重放攻击
com.food.utils.VerifyCodeUtil.clearVerifyCode(phone);
```

### 6. ✅ JWT密钥管理

**问题描述**: JWT密钥硬编码在配置文件中，存在泄露风险。

**修复内容**:
- 修改 `application.yml`，从环境变量 `JWT_SECRET` 读取密钥
- 添加默认值仅用于开发环境
- 在 `JwtUtil` 中添加密钥强度检查（最少32位）
- 启动时输出密钥来源提示

**修改文件**:
- `backend/src/main/resources/application.yml`
- `backend/src/main/java/com/food/utils/JwtUtil.java`

**配置示例**:
```yaml
jwt:
  secret: ${JWT_SECRET:food_wx_secret_key_2024}  # 生产环境必须设置 JWT_SECRET 环境变量
  expiration: 2592000000
```

**环境变量设置**:
```bash
# Linux/Mac
export JWT_SECRET=your_strong_secret_key_here_at_least_32_chars

# Windows PowerShell
$env:JWT_SECRET="your_strong_secret_key_here_at_least_32_chars"
```

## 部署注意事项

### 1. 环境变量配置

生产环境部署前必须设置以下环境变量：

```bash
# JWT密钥（必须设置，至少32位）
export JWT_SECRET=your_strong_random_secret_key_here_32chars

# 数据库密码（建议从环境变量读取）
export DB_PASSWORD=your_database_password

# 阿里云短信服务（如使用）
export ALIYUN_ACCESS_KEY_ID=your_access_key
export ALIYUN_ACCESS_KEY_SECRET=your_access_secret
```

### 2. 数据库更新

需要为现有用户添加默认角色：

```sql
-- 为所有现有用户添加默认角色
UPDATE user SET role = 'USER' WHERE role IS NULL;

-- 设置管理员账号（将第一个用户设为管理员）
UPDATE user SET role = 'ADMIN' WHERE user_id = 1;
```

### 3. 启动验证

启动应用后检查日志输出：

```
=== JwtUtil 配置 ===
expiration: 2592000000ms (30天)
JWT密钥来源: 环境变量 JWT_SECRET  ✓
JWT密钥长度: 48 (符合要求)  ✓
====================
```

如果看到 `警告: JWT密钥使用默认值`，请检查环境变量设置。
